# 测试文件备份目录

## 目录说明

本目录用于存放从 `src/` 目录迁移到 `test/` 目录的原始测试文件备份。

## 备份信息

- **备份时间**: 2026-03-28
- **备份原因**: 测试文件管理规范化迁移
- **备份文件数**: 25个测试文件
- **原始位置**: `src/` 目录下的各个子目录
- **目标位置**: `test/` 目录下的对应位置

## 备份管理策略

### 保留期限
- 备份文件保留30天（从2026-03-28起算）
- 如迁移稳定，可在2026-04-27后清理

### 清理条件
1. 所有迁移后的测试文件正常运行30天
2. 无回滚需求
3. 项目进入下一阶段开发

### 清理命令
```bash
# 谨慎操作：清理备份目录
rm -rf backup_test_files/
```

## 恢复指南

如需从备份恢复测试文件：

1. **检查备份文件**: 确认需要恢复的文件在备份目录中
2. **恢复到原始位置**: 将文件从 `backup_test_files/` 复制回 `src/` 目录的原始位置
3. **更新导入路径**: 恢复后的文件可能需要更新导入路径
4. **验证测试**: 运行恢复后的测试，确保功能正常

## 注意事项

1. **不要直接修改备份文件**: 备份文件应保持原始状态，用于恢复参考
2. **定期检查**: 每月检查备份文件是否仍需保留
3. **项目移交**: 如项目移交，需说明备份目录的作用和清理策略
