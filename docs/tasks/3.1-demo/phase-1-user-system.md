# 3.1 DEMO版 Phase 1: 租户-用户体系

**阶段目标**: 建立租户-用户两层架构，实现多用户登录和权限控制
**预计周期**: 3天  
**开始时间**: 待定  
**依赖阶段**: 无

---

## 任务清单

### Day 1: 用户模型扩展

#### 1.1 用户实体设计
- [ ] 创建用户实体（User Entity）
  - 字段：id, tenantId, username, password, role, email, phone, status
  - 角色枚举：EDITOR(编辑), MANAGER(主管), LEGAL(法务), ADMIN(管理员)
  - 用户与租户关联
- [ ] 更新租户实体，建立与用户的一对多关系
- [ ] 创建用户数据迁移脚本
- [ ] 初始化政务版DEMO用户数据
  - editor@demo-gov / 编辑角色
  - manager@demo-gov / 主管角色
  - legal@demo-gov / 法务角色
  - admin@demo-gov / 管理员角色

#### 1.2 用户服务层
- [ ] 创建UserService
  - 用户CRUD操作
  - 根据租户ID查询用户列表
  - 用户角色验证
- [ ] 创建UserController
  - 用户登录接口（支持按用户名+租户ID登录）
  - 获取当前用户信息接口
  - 用户列表查询接口（管理员权限）

#### 1.3 数据库更新
- [ ] 执行用户表创建SQL
- [ ] 插入DEMO用户数据
- [ ] 验证用户数据正确性

**交付物**:
- 用户实体和服务代码
- 用户表结构
- 4个DEMO用户账号可正常查询

---

### Day 2: 多用户登录与角色权限

#### 2.1 登录页改造
- [ ] 改造前端登录页面
  - 增加租户选择（商务版/政务版）
  - 用户名输入框（支持邮箱格式）
  - 密码输入框
  - 登录角色提示
- [ ] 登录逻辑调整
  - 根据租户类型+用户名查询用户
  - 验证密码和角色
  - JWT token中携带用户ID和角色信息

#### 2.2 角色权限Guard
- [ ] 创建RoleGuard
  - 基于角色控制接口访问权限
  - 支持多角色配置
- [ ] 创建@Roles()装饰器
- [ ] 在关键接口应用权限控制
  - 编辑接口：EDITOR, ADMIN
  - 主管审核接口：MANAGER, ADMIN
  - 法务审核接口：LEGAL, ADMIN

#### 2.3 前端权限控制
- [ ] 创建useAuth hook
  - 获取当前登录用户信息
  - 判断用户角色
  - 权限检查函数
- [ ] 路由权限控制
  - 根据角色动态显示菜单
  - 无权限页面跳转

**交付物**:
- 改造后的登录页面
- 角色权限Guard
- 前端权限控制逻辑

---

### Day 3: DEMO用户切换功能

#### 3.1 用户切换组件
- [ ] 创建用户切换下拉框
  - 显示当前登录用户
  - 支持快速切换其他DEMO用户
  - 切换后刷新页面权限
- [ ] 将切换组件添加到顶部导航栏

#### 3.2 切换逻辑
- [ ] 实现用户切换API
  - 无需重新输入密码
  - 直接切换为指定用户
  - 更新JWT token
- [ ] 切换后状态更新
  - 清空当前用户状态
  - 重新加载用户权限
  - 刷新当前页面

#### 3.3 角色提示
- [ ] 添加当前角色提示
  - 页面顶部显示当前角色名称
  - 显示当前角色可操作权限
  - 审核页面显示当前审核环节

#### 3.4 集成测试
- [ ] 测试4个角色分别登录
- [ ] 测试用户切换功能
- [ ] 测试权限控制正确性
- [ ] 验证三审三校各环节权限

**交付物**:
- 用户切换组件
- 用户切换功能
- 角色提示UI
- 通过集成测试
