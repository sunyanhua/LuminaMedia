# 3.1 DEMO版 政务演示脚本

**版本**: LuminaMedia 3.1 DEMO细节推进版  
**适用环境**: 政务版DEMO环境 (docker-compose本地部署)  
**演示时长**: 15-20分钟  
**目标受众**: 政务客户、政府宣传部门、信息化办公室  
**演示目标**: 展示完整的政务内容创作、审核、发布、监测全流程，突出AI智能生成、三审三校工作流、一键报告等核心价值

---

## 演示环境准备

### 访问信息
- **前端地址**: http://localhost:5174/government
- **后端API**: http://localhost:3003
- **Swagger文档**: http://localhost:3003/api/docs

### DEMO用户账号
政务版DEMO提供4个预设角色账号，密码均为：`LuminaDemo2026`

| 账号 | 角色 | 权限说明 | 显示名称 |
|------|------|----------|----------|
| editor@demo-gov | 编辑 | 创建内容、编辑初审 | 政务编辑 |
| manager@demo-gov | 主管 | 主管复审 | 政务主管 |
| legal@demo-gov | 法务 | 法务终审 | 政务法务 |
| admin@demo-gov | 管理员 | 全部权限 | 系统管理员 |

### 演示前检查清单
- [ ] Docker容器全部正常运行 (`docker-compose ps`)
- [ ] 数据库DEMO数据已初始化 (参考 `scripts/17-demo-government-users-init.sql`)
- [ ] 智慧档案有预置文档和单位画像
- [ ] 参考信息有10-15条政策资讯
- [ ] 公众号有模拟绑定数据和内容
- [ ] 舆情监测有30-50条模拟舆情数据
- [ ] 一键报告功能可正常生成报告

## 演示脚本详细流程

### 1. 开场介绍 (1分钟)
**目标**: 建立信任，明确演示价值

**操作步骤**:
1. 打开演示环境首页
2. 介绍LuminaMedia平台定位

**讲解要点**:
- "大家好，今天我将用15分钟展示LuminaMedia政务版内容营销平台"
- "平台核心价值：通过AI智能技术，帮助政府机构提升宣传效率、保证内容合规、实现数据驱动的精准传播"
- "今天我们将体验从内容创作到发布监测的完整政务工作流"

**预期展示效果**:
- 观众了解平台定位和价值主张
- 建立专业可信的第一印象

### 2. 系统登录 (1分钟)
**目标**: 展示多用户体系和工作台

**操作步骤**:
1. 使用 `editor@demo-gov` / `LuminaDemo2026` 登录政务版
2. 展示工作台概览页面

**讲解要点**:
- "政务版采用租户-用户两层架构，一个单位（租户）下可设置多个用户角色"
- "今天我们使用编辑角色登录，可以看到工作台显示了待办任务、近期内容、数据看板"
- "平台提供四种角色：编辑、主管、法务、管理员，支持完整的审批工作流"

**预期展示效果**:
- 展示登录流程和工作台界面
- 介绍多角色体系的价值

### 3. 智慧档案 (2分钟)
**目标**: 展示知识库管理和单位画像

**操作步骤**:
1. 进入"智慧档案"模块
2. 展示知识库文档列表（政策文件、网页采集）
3. 查看单位画像详情页

**讲解要点**:
- "智慧档案是政务内容创作的基础，包含单位知识库和AI生成的单位画像"
- "知识库支持上传政策文件、采集网页内容，为AI创作提供参考依据"
- "单位画像由AI自动分析生成，包括形象定位、语言风格、视觉偏好、话题偏好、发布习惯五个维度"
- "基于单位画像，AI生成的内容会更符合单位调性和受众预期"

**预期展示效果**:
- 展示知识库管理功能
- 展示AI生成的完整单位画像
- 体现数据驱动的个性化内容策略

### 4. 参考信息 (2分钟)
**目标**: 展示政策资讯自动抓取和一键采用

**操作步骤**:
1. 进入"参考信息"模块
2. 展示自动抓取的政策资讯列表
3. 演示一键采用功能
4. 展示AI修改和润色

**讲解要点**:
- "参考信息模块每日自动抓取最新的政策资讯、行业动态、热点话题"
- "AI会分析资讯与单位的相关度，推荐高相关度的内容"
- "编辑可以一键采用资讯，AI会自动改写为符合单位风格的内容"
- "这个功能帮助编辑快速获取创作灵感，紧跟政策热点"

**预期展示效果**:
- 展示自动抓取的政策资讯列表
- 演示从采用到AI改写的完整流程
- 体现平台的信息获取和智能化处理能力

### 5. 内容创作 (5分钟)
**目标**: 展示AI辅助内容创作完整流程

**操作步骤**:
1. 进入"内容创作"模块，点击"新建内容"
2. 选择内容类型（政务宣传文章）
3. AI生成标题和正文草稿
4. 使用内容编辑器进行修改
5. 手机预览效果
6. 设置发布信息（平台、时间）

**讲解要点**:
- "内容创作采用AI辅助的四步流程：选题→生成→编辑→预览"
- "AI基于单位画像和知识库，生成符合政务风格的内容草稿"
- "编辑器支持富文本编辑、图片插入、格式调整"
- "手机预览功能让编辑实时查看发布效果"
- "发布设置支持选择平台和定时发布"

**关键演示点**:
- AI生成标题和正文的质量
- 编辑器的易用性和功能完整性
- 手机预览的真实效果
- 发布设置的灵活性

**预期展示效果**:
- 展示从空白到完整内容的创作过程
- 体现AI生成内容的质量和效率
- 展示多终端适配能力

### 6. 审核发布 (3分钟)
**目标**: 展示三审三校工作流和用户切换体验

**操作步骤**:
1. 以editor角色提交内容审核
2. 切换为manager角色登录，进行主管复审
3. 切换为legal角色登录，进行法务终审
4. 切换为admin角色登录，演示一键发布
5. 查看发布状态和线上链接

**讲解要点**:
- "政务内容需要严格的三审三校流程：编辑初审、主管复审、法务终审"
- "平台支持多用户角色切换，可以完整模拟实际审批流程"
- "审批过程中，各角色可以添加批注、提出修改意见"
- "终审通过后，管理员可以一键发布到微信公众号等多个平台"
- "发布后可以在平台查看发布状态和线上链接"

**关键演示点**:
- 流畅的用户角色切换体验
- 审批流程的完整性和严谨性
- 一键发布的多平台支持
- 发布状态的可视化监控

**预期展示效果**:
- 展示完整的政务内容审批发布流程
- 体现平台对政府工作流的深度理解
- 展示自动化发布带来的效率提升

### 7. 运营监测 (3分钟)
**目标**: 展示舆情监测、数据看板、一键报告

**操作步骤**:
1. 进入"舆情监测"模块，展示舆情总览和列表
2. 查看舆情详情和情感分析
3. 进入"数据看板"，展示内容表现数据
4. 演示"一键报告"功能，生成舆情周报
5. 展示报告详情和导出功能

**讲解要点**:
- "舆情监测模块实时监控微博、微信、抖音、小红书等平台的政务相关讨论"
- "AI进行情感分析，识别正面、负面、中性舆情"
- "数据看板汇总内容表现数据，包括阅读量、点赞量、分享量等"
- "一键报告功能可以自动生成舆情监测报告、公众号运营报告等"
- "报告支持导出Word格式和生成PPT大纲，方便汇报使用"

**关键演示点**:
- 舆情监测的实时性和全面性
- 情感分析的准确性
- 数据可视化的直观性
- 报告生成的智能化程度

**预期展示效果**:
- 展示从内容发布到效果监测的闭环
- 体现数据驱动的运营优化能力
- 展示智能化报告生成的价值

### 8. 总结 (1分钟)
**目标**: 强化价值主张，引导下一步行动

**操作步骤**:
1. 返回工作台，展示整体数据概览
2. 总结演示亮点

**讲解要点**:
- "刚才我们体验了政务内容从创作到发布监测的完整流程"
- "平台的核心价值体现在三个方面：AI智能提升创作效率、三审三校保证内容合规、数据驱动优化传播效果"
- "对于政府机构来说，平台可以将内容创作时间从几天缩短到几小时，审批流程从线下转移到线上全程可追溯"
- "我们提供政务专属版本，支持私有化部署，确保数据安全合规"

**预期展示效果**:
- 观众对平台价值有清晰认知
- 激发进一步了解和试用兴趣
- 建立专业可信的最终印象

| 开场介绍 | 1分钟 | ±15秒 | 简洁有力，直接切入价值 |
| 系统登录 | 1分钟 | ±15秒 | 快速展示，不纠结细节 |
| 智慧档案 | 2分钟 | ±30秒 | 重点展示单位画像的完整性 |
| 参考信息 | 2分钟 | ±30秒 | 演示一键采用的流畅性 |

| **总计** | **18分钟** | **±4分钟** | **总时长控制在15-20分钟** |

### 重点话术准备
1. **开场话术**: "今天我将展示AI如何帮助政府机构提升宣传效率..."
2. **AI价值话术**: "传统政务内容创作需要查阅大量资料、反复修改，我们的AI基于单位画像和知识库..."
3. **合规话术**: "三审三校是政务内容的生命线，我们通过数字化流程确保每一步都可追溯..."
4. **效率话术**: "从内容创作到发布，传统流程需要3-5天，通过我们的平台可以缩短到3-5小时..."
5. **数据话术**: "不仅关注发布，更关注效果，通过数据看板和舆情监测实现精准优化..."

### 常见问题应对
**Q: AI生成的内容质量如何保证？**
A: AI生成的内容基于单位画像和知识库，确保符合政务风格。所有内容都经过编辑审核和三审三校流程，确保质量。

**Q: 数据安全如何保障？**
A: 政务版支持私有化部署，数据完全独立。采用多重加密、访问控制、操作审计，符合政府信息系统安全要求。

**Q: 是否支持我们现有的审批流程？**
A: 支持自定义审批流程，可以适配各单位的现有流程。平台提供可视化流程配置工具。

**Q: 集成现有政府系统难吗？**
A: 提供标准API接口，支持与OA系统、门户网站等现有系统集成。我们有政府系统集成经验。

### 故障应急预案
1. **网络问题**: 提前准备离线演示素材（截图、录屏）
2. **功能异常**: 跳过故障环节，转场讲解，后续单独演示
3. **登录失败**: 准备备用账号或直接使用预登录状态
4. **生成缓慢**: 提前预生成内容，演示时直接调用
5. **浏览器兼容**: 使用Chrome浏览器，准备备用浏览器
