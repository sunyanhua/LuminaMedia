# LuminaMedia 3.1 DEMO细节推进版 实施规划（修订版）

## 背景与目标

### 当前状态
- 3.0 DEMO版已圆满完成，六阶段全部验收通过，综合评分97分
- 前端dashboard基础框架完整，DEMO演示流程已实现约85%
- 后端核心服务（功能配置、配额限制、演示数据管理）完整实现
- 已支持多租户架构，需扩展多用户及权限体系

### 3.1版本目标
**打造可向政务客户正式演示的完整版本，聚焦微信公众号运营场景**

**核心调整**：
1. 租户-用户两层结构：一个租户下多个用户，审核权限不同
2. 聚焦微信公众号：只演示微信公众号的完整运营流程
3. 内容生产闭环：选题→资料补充→内容生成→审核→发布
4. 政务特色功能：参考信息自动抓取、三审三校用户切换体验

## 第一阶段：租户-用户体系与权限

### 1.1 用户模型扩展

#### 需求说明
- 一个租户（如XX市政务服务中心）下可创建多个用户账号
- 用户角色决定在三审三校中的权限：
  - **编辑**：创建内容、初审
  - **主管**：复审
  - **法务/行政**：终审
  - **管理员**：所有权限

#### 功能需求
| 功能点 | 优先级 | 说明 |
|--------|--------|------|
| 用户登录 | P0 | 同一租户下多用户独立登录 |
| 角色权限 | P0 | 编辑/主管/法务/管理员四种角色 |
| 审核权限控制 | P0 | 不同角色只能看到对应审核环节 |
| 用户切换演示 | P1 | DEMO中可快速切换用户体验完整流程 |

#### 政务版DEMO预设用户
| 用户账号 | 角色 | 用途 |
|----------|------|------|
| editor@demo-gov | 编辑 | 创建内容、初审 |
| manager@demo-gov | 主管 | 复审 |
| legal@demo-gov | 法务 | 终审 |
| admin@demo-gov | 管理员 | 全部权限 |
