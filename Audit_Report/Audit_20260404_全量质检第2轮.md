## 三、新问题发现

### 3.1 🔴 严重: 模块循环依赖导致应用崩溃

**状态**: 🔴 **应用无法启动**  
**错误类型**: `UndefinedModuleException`  
**影响范围**: 整个后端服务

**错误日志**:
```
[Nest] 63  -  04/04/2026, 12:30:32 PM   ERROR [ExceptionHandler] 
UndefinedModuleException [Error]: Nest cannot create the GovernmentModule instance.
The module at index [2] of the GovernmentModule "imports" array is undefined.

Potential causes:
- A circular dependency between modules. Use forwardRef() to avoid it.
- The module at index [2] is of type "undefined". Check your import statements.

Scope [AppModule -> DataAnalyticsModule]
```

**问题根因**:
```
DataAnalyticsModule ──imports──> GovernmentModule
       ▲                               │
       └────────imports────────────────┘
```

**问题文件**:
- `src/modules/data-analytics/data-analytics.module.ts` 第64行
- `src/modules/government/government.module.ts` 第28行

**循环依赖代码**:
```typescript
// data-analytics.module.ts
@Module({
  imports: [
    // ...
    GovernmentModule,  // ← 导入Government
  ],
})

// government.module.ts
@Module({
  imports: [
    // ...
    DataAnalyticsModule,  // ← 导入DataAnalytics（循环！）
  ],
})
```

**修复建议**:
1. **方案A**: 移除循环依赖，提取共享服务到独立模块
2. **方案B**: 使用 `forwardRef()` 解决循环依赖
3. **方案C**: 重构模块边界，合并相关功能

**推荐方案A**:
```typescript
// 创建 shared-marketing.module.ts
@Module({
  imports: [TypeOrmModule.forFeature([MarketingCampaign, MarketingStrategy])],
  providers: [MarketingCampaignRepository, MarketingStrategyRepository],
  exports: [MarketingCampaignRepository, MarketingStrategyRepository],
})
export class SharedMarketingModule {}

// 两个模块都导入SharedMarketingModule，而非互相导入
```

**优先级**: 🔴 **最高 - 阻塞服务启动**
