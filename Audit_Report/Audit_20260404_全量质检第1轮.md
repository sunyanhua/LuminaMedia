## 二、代码质量检查

### 2.1 TypeScript编译检查 🔴

**状态**: 未通过  
**错误数量**: 1个编译错误

```
src/modules/government/services/government-demo.service.ts:77:7 - error TS2353:
Object literal may only specify known properties, and 'isPreset' does not exist in type 
'string[] | Date | number[] | ObjectId | Date[] | ObjectId[] | FindOptionsWhere<MarketingCampaign> | FindOptionsWhere<MarketingCampaign>[]'

77       isPreset: false,
       ~~~~~~~~
```

**根因分析**:
- `government-demo.service.ts` 第77行使用 `marketingCampaignRepository.delete()` 时传入了 `isPreset` 参数
- 但 `delete()` 方法的参数类型 `FindOptionsWhere` 不允许直接传入实体字段进行删除条件匹配
- 需要使用 `find()` 先查询再删除，或使用原生查询

**修复建议**:
```typescript
// 错误代码（第77行附近）
await this.marketingCampaignRepository.delete({
  tenantId,
  isPreset: false,
});

// 应改为
const campaigns = await this.marketingCampaignRepository.find({
  where: { tenantId, isPreset: false }
});
await this.marketingCampaignRepository.remove(campaigns);
```

**优先级**: 🔴 **高 - 阻塞构建**

## 四、安全漏洞扫描

### 4.1 输入验证检查 🔴

**问题**: 缺少全局ValidationPipe

**现状**:
- `main.ts` 中未启用全局 `ValidationPipe`
- 虽然有416个验证装饰器，但没有管道来执行验证

**风险等级**: 🔴 **高**

**修复建议**:
```typescript
// 在 main.ts 中添加
import { ValidationPipe } from '@nestjs/common';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  
  // 启用全局验证管道
  app.useGlobalPipes(new ValidationPipe({
    whitelist: true,        // 移除未定义的属性
    forbidNonWhitelisted: true, // 拒绝未定义的属性
    transform: true,        // 自动转换类型
    transformOptions: {
      enableImplicitConversion: true,
    },
  }));
  
  // ...
}
```

**影响范围**:
- 所有使用 @Body(), @Query(), @Param() 的端点 (88个)
- 可能导致非法数据进入系统

### 4.2 原生查询检查 🟡
