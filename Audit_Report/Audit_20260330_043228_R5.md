## 四、核心问题清单

### 🔴 P0（阻断性问题）

1. **`@qdrant/js-client-rest` 运行时缺失**
   - App容器无法启动，所有API不可用
   - 根因：Docker构建时依赖未正确安装或版本冲突
   - 修复方向：`docker-compose build --no-cache app` 重建，或检查 Dockerfile 依赖安装步骤

2. **`GeoRegion` 实体缺少3个字段**
   - 缺少：`competitionIntensity`、`entryBarriers`、`marketConcentration`
   - 导致 `geo-analysis.service.ts` 和 `region-analysis.service.ts` 大量编译错误
   - 修复方向：在 `geo-region.entity.ts` 中补充这3个字段定义

3. **`geo-analysis` 接口 `never` 类型错误**
   - 接口定义中数组类型约束过严（推断为 `never[]`），导致 `.push()` 的参数类型全部不匹配
   - 修复方向：修正相关接口中的数组字段类型定义

4. **`@nestjs/elasticsearch` 缺失导致4个测试套件失败**
   - 修复方向：`npm install @nestjs/elasticsearch` 或将该依赖加入 package.json

### 🟡 P1（非阻断性问题）

5. **`knowledge-document.service.ts` 类型不匹配**
   - `string` 不可赋值给 `null`（第950行）
   - 修复方向：调整字段类型为 `string | null` 或修正赋值逻辑

6. **`region-analysis.service.ts` 返回对象缺少属性**
   - `strengthsByRegion`、`weaknessesByRegion` 在接口定义中存在但返回值中缺失
   - 修复方向：补充返回对象中的缺失字段
