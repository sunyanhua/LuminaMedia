Claude，之前的 ajv 全局降级导致了 npm run build 崩溃（找不到 codegen 模块）。我们需要撤销全局覆盖，转而修复 ESLint 的兼容性。请按以下步骤执行：
1. 撤销 Overrides：
从 package.json 中移除 ajv 的 overrides（或 resolutions）配置。
2. 升级 ESLint 核心依赖：
运行 npm install eslint@latest @typescript-eslint/eslint-plugin@latest @typescript-eslint/parser@latest --save-dev。
现代版本的 ESLint 插件通常已经解决了 ajv 8.x 的兼容性问题。
3. 处理 ajv-keywords 冲突：
尝试显式安装 ajv 8.x 兼容的关键字库：npm install ajv-keywords@latest ajv@latest --save-dev。
4. 物理清理与重装：
修改完成后，物理删除 node_modules 和 package-lock.json（或 yarn.lock）。
重新运行 npm install。
5. 最终验证（双通才算成功）：
运行 npm run lint：确保不再报 missingRefs 错误。
运行 npm run build：确保不再报 Cannot find module 'ajv/dist/compile/codegen'。
请立即执行，并确保 Build 和 Lint 都能同时通过。
